# PanamaRDDC

Automated road defect detection for Panama using [YOLO11n](https://github.com/ultralytics/ultralytics) trained on the [RDD2022ES](https://www.kaggle.com/datasets/juusos/rdd2022es) dataset, with support for real-world image collection via HEIC/HEIF conversion.

## Usage

All steps are run through `main.py`. The default pipeline order is **1 в†’ 2 в†’ 4 в†’ 3**.

```bash
# Run the full pipeline
python main.py

# Run individual steps
python main.py --steps 1        # Download and format dataset
python main.py --steps 2        # Train model
python main.py --steps 3        # Predict (defaults to val set)
python main.py --steps 4        # Convert HEIC images to JPG

# Common combinations
python main.py --steps 1 2      # Prepare then train
python main.py --steps 4 3      # Convert HEIC photos then predict on them

# Prediction options (step 3)
python main.py --steps 3 --source dataset_yaml/images/test/
python main.py --steps 3 --source path/to/image.jpg
python main.py --steps 3 --source path/to/video.mp4
python main.py --steps 3 --source 0                     # webcam
python main.py --steps 3 --weights path/to/custom.pt
python main.py --steps 3 --conf 0.4 --iou 0.5
python main.py --steps 3 --save-txt --save-conf

# HEIC conversion options (step 4)
python main.py --steps 4 --heic-source path/to/heic/folder
python main.py --steps 4 --heic-source path/to/folder --heic-dest path/to/output/
```

## Pipeline steps

### Step 1 вЂ” Prepare dataset
Downloads RDD2022ES from KaggleHub and splits it 80/20 into `dataset_yaml/images/train` and `dataset_yaml/images/val`. Generates `data.yaml` automatically from `classes.txt` (or by inferring class IDs from the label files if `classes.txt` is absent). Any images missing a label file get an empty `.txt` placeholder so YOLO does not error out. Skipped automatically if `data.yaml` already exists.

### Step 2 вЂ” Train
Downloads `yolo11n.pt` into the project folder if not already present. Trains for 5 epochs at `imgsz=320`, `batch=16`, `workers=0` вЂ” tuned for CPU use on Windows. Weights are saved to `runs/detect/train/weights/`.

### Step 3 вЂ” Predict
Loads `runs/detect/train/weights/best.pt` and runs inference on `dataset_yaml/images/val/` by default, or `dataset_yaml/images/test/` if you have converted Panama images. Annotated results are saved to `runs/detect/predict/`. Fails fast with a clear message if the source folder is empty or weights are missing.

### Step 4 вЂ” Convert HEIC to JPG
Converts HEIC/HEIF images (e.g. from an iPhone) to `.jpg` for use with YOLO. Drop your `.heic` files into `dataset_yaml/images/convert/` and run step 4 вЂ” converted images appear in `dataset_yaml/images/test/`, ready for step 3.

## Notes

- **GPU**: if you have a CUDA-capable GPU, increase `epochs`, `batch`, and `imgsz` in `pipeline/train.py` for faster training and better accuracy.
- **Paths**: all paths are managed centrally in `filehandler.py`. If you need to rename or relocate any folder, that is the only file to edit.
- **Dataset, weights, and runs are git-ignored** вЂ” they are either downloaded automatically or produced by the pipeline.
- **YOLO cache files** (`train.cache`, `val.cache`) inside `dataset_yaml/labels/` are also git-ignored and regenerate automatically.
